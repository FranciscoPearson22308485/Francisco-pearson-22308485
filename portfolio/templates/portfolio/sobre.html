Componentes e fluxos

A arquitetura MVT (Model-View-Template) do Django funciona assim:

  • Model — define a estrutura dos dados (em models.py). Cada classe é uma tabela na base de dados SQLite.
  • View — função em views.py que recebe um pedido HTTP, vai buscar dados ao Model, e devolve uma resposta.
  • Template — ficheiro HTML em templates/ que recebe os dados da View e produz a página final que o utilizador vê.

Fluxo de informação:

  1. O utilizador clica num link no browser (ex: /portfolio/projetos/)
  2. O urls.py faz o routing e mapeia esse caminho para a função projetos_view
  3. A View consulta o Model (Projeto.objects.all()) e busca os dados
  4. A View passa os dados ao Template (projetos.html) com render()
  5. O Template é renderizado em HTML e devolvido ao browser

Modelação em models.py

class Licenciatura(models.Model):
    nome = CharField, sigla = CharField, instituicao = CharField
    ano_inicio = IntegerField, duracao_anos = IntegerField

class Docente(models.Model):
    nome, email, url_pagina_lusofona, foto

class UnidadeCurricular(models.Model):
    nome, sigla, ano_curricular, semestre, ects, imagem
    licenciatura → ForeignKey(Licenciatura)
    docentes → ManyToMany(Docente)

class Tecnologia(models.Model):
    nome, descricao, logo, url_oficial, categoria, nivel_interesse
    tipo → ForeignKey(TipoTecnologia)   # NOVO (Ficha 8)

class TipoTecnologia(models.Model):    # NOVO (Ficha 8)
    nome, descricao
    # Frontend, Backend, Base de Dados, Storage, Outros

class Projeto(models.Model):
    titulo, descricao, data_inicio, data_fim, imagem
    url_github, url_demo, conceitos_uc
    uc → ForeignKey(UnidadeCurricular)
    tecnologias → ManyToMany(Tecnologia)

class TFC(models.Model):
    titulo, autor, orientador, ano, resumo, url, area, destacado
    tecnologias → ManyToMany(Tecnologia)

class Competencia(models.Model):
    nome, descricao, nivel, tipo
    tecnologias → ManyToMany(Tecnologia)
    projetos → ManyToMany(Projeto)

class Formacao(models.Model):
    titulo, instituicao, data_inicio, data_conclusao
    certificado, url, tipo
    tecnologias → ManyToMany(Tecnologia)

class Interesse(models.Model):
    nome, descricao, categoria, url

class MakingOf(models.Model):
    titulo, data, descricao, foto, tipo, entidade_relacionada
    uc → ForeignKey(UnidadeCurricular)
    projeto → ForeignKey(Projeto)

O Git e GitHub trazem várias vantagens ao desenvolvimento:

  • Versionamento — cada commit guarda o estado do projeto, permitindo voltar atrás se algo correr mal.
  • Backup na cloud — o código nunca se perde mesmo que o portátil avarie.
  • Colaboração — possibilidade de trabalhar em equipa via branches e pull requests.
  • Portfolio público — recrutadores podem ver o histórico de commits, o que demonstra ritmo de trabalho.
  • Integração com Codespaces — abre um ambiente Linux pronto a correr Django diretamente do browser.
